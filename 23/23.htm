У исполнителя Калькулятор две команды, которым присвоены номера:

1) прибавь а;

2) умножь b.

Первая из них увеличивает число на экране на а, вторая умножает его на b. Числа  a>0 и b>1 – целые. Программа для Калькулятора – это последовательность команд. Нужно определить, сколько есть программ,которые число М преобразуют в число N (NМ).

     Обозначим количество разных программ для получения числа N из натурального числа M через KN.

     Так как a>0 и b>1, при выполнении любой из команд число увеличивается. Поэтому с помощью такого исполнителя невозможно получить из М число меньшее, чем М.  Следовательно, Кi = 0 для всех  i <M.  Понятно, что для получения числа М из начального значения М существует только одна программа – пустая, не содержащая ни одной команды. Это значит, что       KM=1.

     Теперь нужно построить рекуррентную формулу, связующую KN с предыдущими элементами последовательности К1, К2, …, КN-1.

     Если число N не делится на b, то на последнем шаге оно может быть получено только операцией сложения, в этом случае KN = KN-a.

     Если же N делится на b, то последней командой может быть, как сложение, так и умножение. Поэтому нужно сложить KN – a (количество программ с последней командой сложения) и KN/b  (количество программ с последней командой умножения). В итоге получаем:
